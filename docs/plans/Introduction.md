I am a vision scientist studying how the primate brain processes visual information. I have noticed that many labs use different methods for generating visual stimuli using digital displays. The most commonly used program is Psychtoolbox, which is an open-source software package for MATLAB. There are other programs, such as PsychoPy, which is a Python-based software package for generating visual stimuli. I have also seen some labs use custom-built software written in C or other programming languages. Vision science has traditionally used simple stimuli such as gratings, bars, dots, and static images to study visual processing. However, there is a growing interest in using more complex and naturalistic stimuli, such as videos, virtual reality environments, and real-world scenes. These stimuli may provide new insight into visual processing and navigation, but they also present new challenges for stimulus generation and presentation. Some labs have pivoted to using Unity, a game engine that allows for the creation of complex and interactive visual stimuli. However, there are currently no options available that allow an experienced programmer full access to the underlying graphics hardware, which is necessary for achieving the highest possible performance and reproducibility in vision science experiments. This gap was the motivation for developing the VisionStimulusEngine (VSE), a new software package for generating visual stimuli using the Vulkan graphics API in the Rust programming language. VSE is a new project that aims to meet the following goals:

1. Provide access to the underlying graphics API calls for advanced users, while also providing a high-level interface for users who are learning graphics programming. The goal is to allow users to incrementally learn graphics programming by starting with a simple interface and gradually exposing more advanced features as they become more comfortable with the software. This approach is intended to make it easier for vision scientists to learn graphics programming and to encourage them to explore the full capabilities of their graphics hardware.

2. Have (at least) millisecond accurate timing precision for visual stimulus presentation. This is imperative for vision science experiments, where precise timing is often necessary to accurately measure neural responses to visual stimuli.

3. Allow for full reproducibility of programatically generated, rich visual stimuli. Reproducibility is critical for image-computable models of neural responses. 

4. Where possible, adhere to a similar API design to Psychtoolbox, which is widely used in the vision science community. This will make it easier for users who are familiar with Psychtoolbox to transition to VSE and to use it in their experiments. By providing a familiar interface, we hope to encourage more vision scientists to adopt VSE and to take advantage of its advanced features.
